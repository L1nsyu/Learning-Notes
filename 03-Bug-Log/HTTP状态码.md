# 🌐 HTTP 状态码快速排查指南：如何通过数字定位 Bug？

在前后端联调时，通过观察接口返回的 3 位数状态码，我们可以迅速锁定问题发生的位置。

## 💡 核心法则：看“第一位数字”定大方向

记住这个口诀，就能解决 80% 的定位问题：

*   **`1xx` (信息性状态码)**：服务器已接收请求，正在处理。（日常开发极少遇到，可忽略）
*   **`2xx` (成功状态码)**：**一切正常**。兄弟们，代码没毛病！
*   **`3xx` (重定向状态码)**：**位置变了**。你要找的东西搬家了，服务器让你去新地址找。
*   **`4xx` (客户端错误)**：**前端/用户的锅**。你发过去的请求有问题（地址填错、参数传错、没权限等），后端看不懂或拒绝处理。
*   **`5xx` (服务器错误)**：**后端的锅**。请求完美送达，但后端自己的代码写炸了，或者数据库挂了。

## 🔍 高频状态码详解与排查动作

### 🟢 2开头：成功 (Success)
*   **`200 OK`**
    *   **含义**：请求完美成功，数据正常返回。
    *   **排查**：无需排查，喝杯咖啡庆祝一下。

### 🟠 4开头：前端/客户端的锅 (Client Error)
> **排查方向**：检查你的前端代码、Apifox 请求地址、请求参数、Headers、请求方式（GET/POST）。**不要去改后端代码逻辑！**

*   **`400 Bad Request` (错误请求)**
    *   **含义**：前端传给后端的参数格式不对。比如后端要 `Integer`，前端传了字母；或者 JSON 格式缺了逗号。
    *   **排查**：仔细核对 Apifox 里的 `Params` 或 `Body` 参数，和后端接口文档要求的参数名、类型是否完全一致。
*   **`401 Unauthorized` (未授权)**
    *   **含义**：你没有登录，或者你的 Token/Session 过期了，服务器不认识你。
    *   **排查**：检查请求头（Headers）里有没有带上正确的 Token (如 `Authorization: Bearer xxx`)。
*   **`403 Forbidden` (禁止访问)**
    *   **含义**：你登录了，服务器认识你，但是**你没有权限**做这件事（比如普通用户想点删除按钮）。
    *   **排查**：检查当前账号的角色权限配置。
*   **`404 Not Found` (未找到)**
    *   **含义**：你请求的 URL 地址（门牌号）根本不存在。
    *   **排查**：**拼写检查！** 看看 Apifox 里的 URL 是不是多打了一个字母，或者后端根本就没写这个路径的 Controller。
*   **`405 Method Not Allowed` (方法不被允许)**
    *   **含义**：URL是对的，但请求方式用错了。比如后端是 `@PostMapping`，你用 `GET` 或 `PUT` 去请求。
    *   **排查**：核对 Apifox 下拉框里的 `GET/POST/PUT/DELETE` 是否和后端 Controller 里的注解严格对应。

### 🔴 5开头：后端的锅 (Server Error)
> **排查方向**：前端请求没问题，是后端服务器内部崩了。**立刻去打开 IDEA（或服务器），查看控制台的红色报错日志（Exception）！**

*   **`500 Internal Server Error` (内部服务器错误)**
    *   **含义**：最常见的后端报错。后端代码在执行时发生了未捕获的异常（如空指针异常 `NullPointerException`、SQL 语法写错、数组越界等）。
    *   **排查**：看 IDEA 控制台报错日志，顺藤摸瓜找到具体是哪一行 Java 代码报错，去修复逻辑。
*   **`502 Bad Gateway` (网关错误)**
    *   **含义**：通常作为代理的服务器（如 Nginx）正常，但背后的真实后端服务器（如 Tomcat/Spring Boot）挂了或者没启动。
    *   **排查**：检查你的 Spring Boot 项目是不是忘记点“运行”了，或者项目是不是自己停掉了。
*   **`504 Gateway Timeout` (网关超时)**
    *   **含义**：后端处理太慢了！前端等得花儿都谢了（通常超过 30 秒或 60 秒）还没返回结果。
    *   **排查**：检查后端代码是不是死循环了、SQL 查询是不是太慢没加索引，或者在请求第三方接口时卡住了。

## 📝 总结：快速排查速查表

| 状态码 | 谁的问题？ | 常见原因 | 你的第一反应是去查哪里？ |
| :--- | :--- | :--- | :--- |
| **200** | 没问题 | 成功 | 继续开发下一个功能 |
| **400** | 💻 前端 | 参数传错、格式不对 | 查 Apifox 的 Body/Params 数据 |
| **401/403** | 💻 前端 | 没登录 / 没权限 | 查 Token 和账号权限 |
| **404** | 💻 前端 | 路径拼写错误 | 对比前后端的 URL 地址是否一字不差 |
| **405** | 💻 前端 | GET/POST 用混了 | 检查 `@xxxMapping` 和请求方法是否匹配 |
| **500** | 🖥️ 后端 | 代码抛出异常、SQL 报错 | **立刻去看 IDEA 控制台的红字日志** |
| **502** | 🖥️ 后端 | 后端服务没启动/宕机 | 检查 Spring Boot 是否正在运行 |
