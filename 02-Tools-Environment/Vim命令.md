## 01.什么是 Vim？

- **一个在终端里运行的文本编辑器**：它没有图形界面，所有的操作都通过键盘完成。
- **Git 的默认编辑器**：当你执行某些 Git 命令（比如 git commit 不带 -m 参数，或者像你遇到的 git pull 后需要合并）时，Git 会自动为你打开 Vim，让你输入或确认信息。
- **强大但学习曲线陡峭**：Vim 以其极高的效率和强大的功能而闻名，但它的操作逻辑和我们平时用的记事本、Word 完全不同，所以初学者会感到困惑。

### Vim 最核心的概念：模式 (Modes)

理解了“模式”，你就理解了 Vim 的一半。Vim 主要有三种你必须知道的模式：

1. **Normal Mode (正常模式)**
    - 这是你**一进入 Vim 时的默认模式**。
    - 在这个模式下，你按下的**几乎所有按键都不会输入文字**，而是会被当作**命令**来执行（比如移动光标、删除行、复制粘贴）。
    - 这是 Vim 的“指挥中心”。
    - **无论你处在什么模式，按 Esc 键，总能回到正常模式。** 这是最重要的救命键！
2. **Insert Mode (插入模式)**
    - 这个模式就和我们平时用的记事本一样，**按键即输入文字**。
    - 在你的截图中，左下角显示的 -- INSERT -- 就表示你正处于插入模式。
    - 如何进入？在**正常模式**下按 i (insert) 键即可。
3. **Command-Line Mode (命令行模式)**
    - 这个模式用来执行一些更复杂的操作，比如保存、退出、搜索、替换等。
    - 如何进入？在**正常模式**下按冒号 : 键。光标会立刻跳到屏幕左下角，等待你输入命令。
    - 你问的 :wq 就是这个模式下的命令。

## 02.Vim的一些基本指令。

### 像 :wq 一样常用的 Vim 命令（“生存”指南）

记住下面这些，你就能在 Git 弹出的 Vim 界面中轻松生存下来。所有这些命令都是在**正常模式 (Normal Mode)**下按下的。

#### 1. 保存和退出 (最重要的！)

| 命令    | 作用                               | 解释                                                         |
| ------- | ---------------------------------- | ------------------------------------------------------------ |
| **:wq** | **保存并退出 (Write & Quit)**      | 这是最常用的，完成编辑后用它。                               |
| :q      | 退出 (Quit)                        | 如果文件没有被修改，可以直接退出。                           |
| **:q!** | **强制退出 (Quit without saving)** | 如果你改错了东西，想反悔，就用这个！它会放弃所有修改并退出。 |
| :w      | 仅保存 (Write)                     | 只保存文件，但不退出编辑器。                                 |

#### 2. 移动光标 (在正常模式下)

| 按键     | 作用                     |
| -------- | ------------------------ |
| j 或 ↓   | 向下移动一行             |
| k 或 ↑   | 向上移动一行             |
| h 或 ←   | 向左移动一个字符         |
| l 或 →   | 向右移动一个字符         |
| gg       | 移动到文件的**第一行**   |
| G (大写) | 移动到文件的**最后一行** |

#### 3. 编辑文本 (在正常模式下)

| 命令   | 作用                           | 解释                                     |
| ------ | ------------------------------ | ---------------------------------------- |
| **i**  | **进入插入模式 (Insert Mode)** | 在光标当前位置开始输入文字。             |
| a      | 进入插入模式                   | 在光标**之后**的位置开始输入文字。       |
| o      | 进入插入模式                   | 在当前行的**下方**新建一行，并开始输入。 |
| **dd** | **删除当前行**                 |                                          |
| yy     | **复制当前行** (Yank)          |                                          |
| p      | **粘贴** (Paste)               | 把之前复制或删除的内容粘贴到光标下一行。 |
| x      | 删除光标所在的单个字符         |                                          |
| **u**  | **撤销 (Undo)**                | 非常有用，撤销上一步操作。               |

### 总结一下你的操作流程

当你再次遇到 Git 弹出 Vim 的情况时，你应该这么想：

1. **“我现在在 Vim 里，默认是正常模式。”**
2. **“我需要写提交信息吗？”**
    - **需要**: 按 i 进入插入模式，然后像普通编辑器一样打字。打完后，按 Esc 回到正常模式。
    - **不需要** (比如 Git 已经提供了默认信息): 直接进行下一步。
3. **“我已经完成了，需要保存退出了。”**
    - 在正常模式下，输入 :wq，然后按回车。
4. **“哦不，我搞砸了，想取消这次提交。”**
    - 在正常模式下，输入 :q!，然后按回车。





在 Bash（终端）中，复制和粘贴的方式与我们在 Word 或浏览器中习惯的 Ctrl+C / Ctrl+V **不一样**

### 1. 如果你在 Windows 上使用 Git Bash (MINGW64)

- **复制 (Copy):**
    - **方法一（鼠标）**：用鼠标左键**选中**你要复制的文字，它通常会自动复制（或者选中后按一下 **Enter** 键）。
    - **方法二（快捷键）**：选中文字后，按下 **Ctrl + Insert**。
    - **方法三（右键）**：选中文字 -> 鼠标右键 -> Copy。
- **粘贴 (Paste):**
    - **方法一（快捷键 - 最推荐）**：按下 **Shift + Insert**。
    - **方法二（鼠标）**：在光标处点击鼠标**右键** -> Paste。
    - **方法三（中键）**：如果你的鼠标有滚轮，直接按下**鼠标滚轮（中键）**通常也可以粘贴。

### 2. 如果你在 Linux (Ubuntu/CentOS 等) 或 Windows Terminal (WSL)

大多数现代 Linux 终端模拟器为了避免冲突，都加了一个 Shift 键。

- **复制**：**Ctrl + Shift + C**
- **粘贴**：**Ctrl + Shift + V**
