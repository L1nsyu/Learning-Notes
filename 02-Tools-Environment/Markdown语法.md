# 📝 Markdown 语法速查指南 (Markdown Cheat Sheet)

> **创建时间**：2026-03-03
> **标签**：#Markdown #文档排版 #工具

Markdown 是一种轻量级标记语言，它允许我们使用易读易写的纯文本格式编写文档，也是我们维护这个知识库的核心语言。

---

## 一、 基础排版语法 (Basic Syntax)

### 1. 标题 (Headings)
在文字前面加上 `#` 和一个空格。`#` 的数量代表标题的级别（最多支持 6 级）。
- **写法**：
  `# 这一级标题最大`
  `## 这是二级标题`
  `### 这是三级标题`

### 2. 文本强调 (Emphasis)
- **加粗**：用两个星号包围。例如 `**重点内容**` 渲染为 **重点内容**。
- **斜体**：用一个星号包围。例如 `*特定名词*` 渲染为 *特定名词*。
- **删除线**：用两个波浪号包围。例如 `~~作废代码~~` 渲染为 ~~作废代码~~。
- **\<br>** :换行。
- **\\** : 转义符号。

### 3. 列表 (Lists)
- **无序列表**：使用 `-` 或 `*` 加空格。
  `- 第一项`
  `- 第二项`
- **有序列表**：使用数字加英文句号和空格。
  `1. 第一步`
  `2. 第二步`

### 4. 代码引用 (Code)
- **行内代码**：用反引号 (Esc 键下方) 包围专业词汇或短代码。
  例如：修改 `` `pom.xml` `` 文件。

- **多行代码块**：用三个反引号包围，并在第一行末尾指定编程语言（如 java, sql, xml）以实现语法高亮。
  ```java
  public class HelloWorld {
      System.out.println("Hello");
  }

### 5. 链接与图片 (Links & Images)
   超链接：格式为 [显示的文字](真实的网址)。
   例如：[访问 Maven 官网](https://maven.apache.org/)

   插入图片：和超链接很像，只是前面多了一个感叹号 !。格式为 ![图片加载失败时的提示文字](图片路径)。
   例如：![404报错截图](../assets/404-error.png)

### 6. 其他常用格式
- 引用区块：行首加上大于号 > 和空格。常用于高亮提示或记录控制台报错。
    > 这是一个重要的注意项！

- 分割线：使用三个以上的连字符 --- 或星号 ***，用于隔开不同段落。

## 二、 进阶语法：如何绘制表格 (Tables)
在 Markdown 中绘制表格，主要是用竖线 | 和短横线 - 拼出来的。

1. 表格规则解析
-  | (竖线)：用来分隔每一列。
-   \- (短横线)：写在第二行，用来分隔表头和下面的数据。
(这里双引号是为了不然短横线进行作用)
-  : (冒号)：加在第二行的短横线旁边，用来控制对齐方式：
-  :--- ：左对齐（默认）
-  :---: ：居中对齐
-  ---: ：右对齐


2. 真实案例实战

🔍 Markdown 源码写法：

| 特性         | @SpringBootTest (集成测试)                   | 单元测试 (Unit Test)                             |
| :----------- | :------------------------------------------- | :----------------------------------------------- |
| **目标**     | 测试整个应用或多层组件的协作                 | 只测试一个孤立的类或方法                         |
| **环境**     | 加载**完整**的 Spring 环境                   | **不加载** Spring 环境，通常用 Mock 对象替代依赖 |
| **速度**     | **慢** (因为要启动整个应用)                  | **极快**                                         |
| **何时使用** | 当你需要测试端到端的功能或确保各层配合无误时 | 当你只想验证某个类的内部逻辑是否正确时           |


## 三、 避坑指南 (Tips)
- 空格极其重要：Markdown 对空格很敏感。不管是标题的 # 还是列表的 -，符号后面必须跟一个空格，否则无法正常渲染。
- 特殊字符转义：如果你想在文档里真实地打出一个 * 号，而不是让它变成斜体，可以在它前面加一个反斜杠 \ 进行转义，例如 \*。
